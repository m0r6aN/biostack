WHAT WAS BUILT

A complete C# ASP.NET Core backend for BioStack Mission Control with:

ARCHITECTURE
  - Clean layered design (Domain, Application, Infrastructure, API)
  - Dependency injection throughout
  - Repository pattern with generics
  - Service-oriented business logic
  - Minimal API endpoints

DOMAIN ENTITIES (7)
  1. PersonProfile - User with biometrics and goals
  2. CompoundRecord - Tracked substances with lifecycle
  3. CheckIn - Daily health metrics (weight, sleep, energy, recovery, mood, GI)
  4. ProtocolPhase - Named protocol periods
  5. TimelineEvent - Unified event aggregation
  6. KnowledgeEntry - Educational compound information
  7. InteractionFlag - Overlap detection results

ENUMERATIONS (7)
  - CompoundCategory (5 values)
  - CompoundStatus (5 values)
  - SourceType (4 values)
  - EventType (8 values)
  - Sex (4 values)
  - EvidenceTier (4 values)
  - OverlapType (4 values)

VALUE OBJECTS (1)
  - CalculatorResult with factory method

CONTRACTS (18 total)
  - 10 Request DTOs
  - 8 Response DTOs
  All immutable records

SERVICES (8)
  1. ProfileService - CRUD for profiles
  2. CompoundService - CRUD for compounds
  3. CheckInService - Create/list check-ins
  4. ProtocolPhaseService - Create/list phases
  5. TimelineService - Unified timeline
  6. CalculatorService - 3 calculators with disclaimers
  7. KnowledgeService - Query knowledge base
  8. OverlapService - Pathway intersection detection

REPOSITORIES (6)
  - PersonProfileRepository
  - CompoundRecordRepository
  - CheckInRepository
  - ProtocolPhaseRepository
  - TimelineEventRepository

  All with specific queries and proper ordering

API ENDPOINTS (21 total)

  Compounds:       4 endpoints (GET profile, POST create, PUT update, DELETE)
  Check-Ins:       2 endpoints (GET profile, POST create)

  Health Check:    1 endpoint  (/health)
  Swagger/OpenAPI: 1 endpoint  (/swagger)

CALCULATORS (3)
  1. Reconstitution: (Peptide mg × 1000) / (Diluent mL × 100) → mcg/unit
  2. Volume: Desired Dose / Concentration → mL
  3. Conversion: Amount × ConversionFactor → New Unit
  All with mandatory disclaimer: "This is a mathematical calculation only. Not medical advice."

BUILD & RUN INSTRUCTIONS
================================================================================

Prerequisites:
  .NET 8 SDK (no other dependencies required)

Restore:
  dotnet restore

Build:
  dotnet build

Run Tests:
  dotnet test

Start Server:
  cd src/BioStack.Api
  dotnet run

Access API:
  HTTP:    http://localhost:5000
  HTTPS:   https://localhost:5001
  Swagger: http://localhost:5000/swagger

Example Request (Create Profile):
  POST /api/v1/profiles
  {
    "displayName": "John Doe",
    "sex": 1,
    "weight": 75.5,
    "goalSummary": "Optimize health",
    "notes": "Test profile"
  }

Example Response (Calculator):
  POST /api/v1/calculators/reconstitution
  Input:  {"peptideAmountMg": 5, "diluentVolumeMl": 2.5}
  Output: {
    "input": 5.0,
    "output": 2000.0,
    "unit": "mcg/unit (0.01mL)",
    "formula": "Concentration = (Peptide mg * 1000) / (Diluent mL * 100)",
    "disclaimer": "This is a mathematical calculation only. Not medical advice."
  }
